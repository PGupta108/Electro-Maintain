## 🚀 HOW TO RUN LOCALLY (Step by Step)

### Step 1 — Get MongoDB (Free)
1. Go to → https://www.mongodb.com/atlas
2. Sign up → Create Free Cluster (M0 Free Tier)
3. Create a Database User (username + password)
4. Under "Network Access" → Add `0.0.0.0/0` (allow all IPs)
5. Under "Connect" → Drivers → Copy connection string
   → Looks like: `mongodb+srv://user:pass@cluster0.abc.mongodb.net/electromaintain`

### Step 2 — Setup Backend
```bash
cd backend
npm install
cp .env.example .env
```

Now open `.env` and fill in:
```
MONGODB_URI=<your Atlas connection string>
JWT_SECRET=anyrandomlongstring123abc
EMAIL_USER=yourgmail@gmail.com
EMAIL_PASS=<Gmail App Password — see Step 3>
ANTHROPIC_API_KEY=<optional — leave blank to use knowledge base only>
```

```bash
npm run dev
# Server starts at http://localhost:5000
# You should see: ✅ MongoDB connected
```

### Step 3 — Gmail App Password (for email reminders)
1. Go to → https://myaccount.google.com/security
2. Enable 2-Step Verification
3. Go to → https://myaccount.google.com/apppasswords
4. Select "Mail" → "Other (Custom name)" → type "ElectroMaintain"
5. Copy the 16-character password → paste into `EMAIL_PASS` in `.env`

### Step 4 — Setup Frontend
```bash
cd frontend
npm install
npm run dev
# Opens at http://localhost:3000
```

### Step 5 — Open and Use
1. Visit http://localhost:3000
2. Register a new account
3. Click **Devices** → **+ Add Device**
4. Fill in your device details and submit
5. Watch the automatic maintenance reminders appear in **Reminders** tab!

## 🤖 SMART AUTO-REMINDER ENGINE EXPLAINED

**File:** `backend/services/smartReminders.js`

### How it works (3 steps):

**Step 1 — Knowledge Base**
A built-in database of maintenance tasks per device category. Like a digital device manual:
- Laptop → Fan cleaning every 90 days, Battery check every 60 days, OS update every 30 days...
- Vehicle → Oil change every 90 days, Tyre check every 30 days...
- AC → Filter cleaning every 30 days, Annual service every 365 days...
- (8 categories total, 4–9 tasks each)

**Step 2 — Smart Interval Adjustment**
The base intervals are adjusted for each specific device using 4 factors:
```
Multiplier = conditionFactor × ageFactor × usageFactor × warrantyFactor

conditionFactor: Excellent=1.3x (less frequent), Poor=0.5x (more frequent)
ageFactor:       >5 years=0.7x, <6 months=1.2x
usageFactor:     12+ hrs/day=0.7x, ≤2 hrs/day=1.2x
warrantyFactor:  Expired=0.85x (more careful)
```

**Step 3 — Optional AI Enhancement**
If `ANTHROPIC_API_KEY` is set in `.env`, the engine also calls Claude AI to generate
2–3 device-specific reminders based on the exact brand/model (known issues, manufacturer tips).
If the key is not set, the system works perfectly with the knowledge base alone.

## 🔬 ML ENGINE EXPLAINED (Honest)

**File:** `backend/routes/ml.js`

### What it is:
Pure JavaScript algorithms — NOT a neural network or trained model.
These are real statistical/algorithmic techniques used in data science.

### Algorithm 1: Weighted Moving Average (Date Prediction)
```
1. Collect all past completion intervals for device + task type
2. Assign weight i+1 to the i-th interval (recent = higher weight)
3. WMA = Σ(interval × weight) / Σ(weights)
4. Predicted date = last completion date + WMA days
5. Confidence = f(variance, sample count) → 0.40 to 0.95
```
Needs ≥2 completions to activate. Until then, uses defaults.

### Algorithm 2: Priority Scoring
```
Score = (TaskUrgency×25) + (CategoryRisk×20) + (DeviceAge×15) +
        (UsageHours×15) + (OverdueHistory×15) + (WarrantyExpired×10) +
        (DaysUntilDue bonus×0-10)
Max = 100
```

### Algorithm 3: Anomaly Detection (5 rules)
```
Rule 1: ≥3 overdue tasks on same device → "Neglected device"
Rule 2: Same task type within 7 days → "Duplicate reminder"
Rule 3: Condition = Poor → "Immediate inspection needed"
Rule 4: Usage ≥10h/day + no maintenance in 30 days → "High-risk neglect"
Rule 5: Warranty expired + no inspection pending → "Post-warranty gap"
```

### Is it "real ML"?
Honest answer: WMA is a genuine statistical technique used in time-series forecasting.
Priority scoring is a multi-factor model (similar to linear regression with fixed weights).
Anomaly detection is rule-based (expert system). Not deep learning — but real, useful, and
explainable, which is important for a PBL project and research paper.
